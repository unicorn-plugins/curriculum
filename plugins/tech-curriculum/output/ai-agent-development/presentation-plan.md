## WHY - 프리젠테이션의 목적과 핵심 메시지

### 1. 교육의 본질적 가치

**"AI Agent 개발의 전체 여정을 5일 만에 경험할 수 있습니다"**

#### 핵심 가치 제안

1. **실무 즉시 적용**
   - LLM API부터 MAS까지 프로젝트에 바로 쓸 수 있는 기술 습득
   - 5일간 학습 내용을 종합한 통합 프로젝트로 실전 역량 확보
   - 교육 종료 후 2주 내 실무 적용 가능

2. **체계적 성장 경로**
   - 단계별 여정: 대화 → 데이터 → 도구 → 지식 → 에이전트
   - 각 단계가 다음 단계의 기반이 되는 점진적 학습
   - 초보자도 따라올 수 있는 명확한 로드맵

3. **최신 기술 반영**
   - 2025~2026년 최신 트렌드 반영 (LangChain 1.0, MCP, GraphRAG)
   - 멀티 LLM 비교 (OpenAI, Claude, Gemini, Groq)
   - 산업 표준 도구 및 프레임워크 활용

### 2. 차별화 포인트

**"이론만 배우지 않습니다. AI가 개발하고, 여러분은 이해합니다"**

#### 3가지 차별성 요소

1. **실습 중심 (50% 이상)**
   - 전 과정 실습 비중 50% 이상
   - 매 모듈마다 실제 동작하는 코드 작성
   - Claude Code가 개발한 코드를 직접 실행하며 학습

2. **멀티 LLM 비교 학습**
   - OpenAI, Claude, Gemini, Groq 4개 LLM 동일 시나리오 비교
   - 각 LLM의 강점과 특성 이해
   - 프로젝트 상황에 맞는 LLM 선택 역량 확보

3. **핵심개념 도식화**
   - SAS 패턴, RAG 파이프라인, Agent Flow 등 복잡한 개념을 아키텍처 다이어그램으로 명확 전달
   - 시각적 자료로 이해도 향상
   - 교재와 함께 제공되는 다이어그램으로 복습 용이

## HOW - 전달 방식과 구성 전략

### 1. 슬라이드 구성 원칙

#### 핵심 원칙
- **한 페이지 = 하나의 핵심 메시지** (최대 3개 핵심 포인트)
- **시각적 여정 표현**: 5단계 성장 경로를 이모지와 화살표로 표현
- **실습 중심 강조**: 각 Day별 핵심 실습을 스크린샷/다이어그램으로 시각화

#### 정보 계층 구조
1. **타이틀** (22pt, 딥퍼플): 슬라이드의 핵심 메시지
2. **서브 타이틀** (18pt): 부가 설명 또는 컨텍스트
3. **본문** (16pt): 주요 내용, 리스트
4. **캡션** (14pt): 부가 정보, 출처, 참고사항

### 2. 스타일 가이드

#### 테마: Modern
- **특징**: 깔끔한 여백, 세련된 느낌, 미니멀 디자인
- **레이아웃**: 충분한 공백, 명확한 계층 구조
- **요소**: 플랫 디자인 아이콘, 심플한 도형

#### 컬러 팔레트
- **강조색 (Primary)**: 딥퍼플 (#673AB7)
  - 사용: 제목, 중요 키워드, 아이콘, 강조 박스
- **배경색 (Background)**: 화이트 (#FFFFFF)
  - 사용: 슬라이드 배경 (가독성 우선)
- **보조색 (Secondary)**: 라이트 그레이 (#F5F5F5)
  - 사용: 구분선, 배경 박스
- **텍스트색 (Text)**: 다크 그레이 (#333333)
  - 사용: 본문 텍스트

#### 폰트
- **서체**: 맑은고딕
- **크기 규격**:
  - 제목: 22pt (Bold)
  - 부제목: 18pt (SemiBold)
  - 본문: 16pt (Regular)
  - 캡션: 14pt (Regular)

### 3. 전개 방식

#### 5단계 스토리텔링

1. **흥미 유발** (슬라이드 1~2)
   - 표지에서 5단계 여정 제시
   - 성장 경로 다이어그램으로 전체 그림 제공

2. **가치 공감** (슬라이드 3~4)
   - 왜 AI Agent 개발을 배워야 하는지 (비즈니스 가치)
   - 3가지 차별화 포인트로 신뢰 구축

3. **구체적 경로** (슬라이드 5~12)
   - Day별 학습 내용과 실습 구성
   - 섹션 전환 슬라이드로 명확한 구분
   - 세부 내용 슬라이드로 구체성 제공

4. **신뢰 구축** (슬라이드 13)
   - 실습 비중 표로 증거 제시
   - Claude Code 활용으로 차별성 강조

5. **행동 유도** (슬라이드 14~15)
   - 통합 프로젝트 및 평가 방법 안내
   - 5일 후 기대 효과로 동기 부여
